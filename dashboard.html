Archived research artifact

System status content is not a public proof source

The former status page mixed local health checks with fallback display values. That can read as live operational proof even when no service is reachable, so the public artifact is retired until a verifier-backed status surface exists.

Current posture

Public availability, model-runtime, source/proof, and court-workflow readiness claims must come from verifier output produced in the current release context.

Replacement criteria

  • Show timestamps and command names for each proof item.
  • Display unavailable states instead of fallback success values.
  • Separate internal diagnostics from public product claims.
  • Keep readiness blockers visible until the verifier clears them.

What remains public

The homepage shows product states, boundaries, and human-review locks without implying that a live status service is exposed from the public site.

Boundary

No production filing, service, payment, provider upload, or court contact is available from this website. Status copy cannot override that boundary.
